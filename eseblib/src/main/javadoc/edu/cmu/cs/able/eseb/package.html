The eseblib contains code to start an event bus server and clients that can connect to the event bus to publish events or subscribe to events. Events published may have several types which include basic data types and complex (composite) data types. All data types are instances of the DataType class. The following classes define the data types that can be send through the event bus (each event corresponds to a single data type):

  • DataType: Superclass for all data types. This is an abstract class that does not implement any particular data type but rather all common functionality of all data types.
  • DataTypeCode: Each data type has a unique code which is obtained from this enumeration.
  • Int32DataType: Data type containing a 32-bit integer (maps to int in Java).
  • Int64DataType: Data type containing a 64-bit integer (maps to long in Java).
  • ListDataType: Data type containing a list of other data types, not necessarily equal. The list can contain no values or can contain values from many different types including more complex data types.
  • RecordDataType: Data type mapping strings (names) to data types.
  • StringDataType: Data type containing a string (maps to a String in Java).
  • TimeDataType: Data type containing time with up to microsecond precision (maps to a Date in Java). Note that dates in Java only have millisecond precision so extra precision is lost when reading the data type.

Clients connect to the event bus using the BusConnection class. Each server opens two ports, one for publishing and one for subscribing. Each bus connection may connect to both or either. The BusServer implements the server. If the connection between the client and the server is lost, the client will keep trying to connect.

Several other classes are defined that allow input and output of data types in a stream-like way:

  • BufferedDataTypeInputStream: Inputs data types with buffering.
  • BufferedDataTypeOutputStream: Outputs data types with buffering.
  • DataTypeInputStream: Abstract interface for streams that input data types.
  • DataTypeInputStreamImpl: Basic implementation of a stream that inputs data types.
  • DataTypeOutputStream: Abstract interface for streams that output data types.
  • DataTypeOutputStreamImpl: Basic implementation of a stream that outputs data types.
  • DualSocketDataTypeStream: Implementation of a InOutDataTypeStream that uses sockets for both input and output.
  • InOutDataTypeStream: Stream that has both an input and an output channel. Both channels are kept synchronized: if one fails, both are closed.
  • InvalidDataFormatException: Exception thrown when the data received in an input stream cannot be converted to a data type.
